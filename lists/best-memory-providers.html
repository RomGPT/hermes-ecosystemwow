breakdown

mem0ai / mem0

As the official memory provider for Hermes Agent, Mem0 offers a universal layer that combines semantic, keyword, and entity-based matching for multi-signal retrieval. It utilizes temporal reasoning to rank memories by relevance and features a single-pass extraction process that accumulates facts without overwriting existing data.

garrytan / gbrain

This opinionated memory system features a self-wiring knowledge graph that extracts entity relationships without requiring LLM calls. It employs a hybrid search strategy that boosts vector retrieval with backlink-based graph ranking and supports 14 different embedding providers.

vectorize-io / hindsight

Hindsight implements a 'retain, recall, and reflect' workflow designed for long-term agent learning and enterprise-grade storage via PostgreSQL or Oracle. It simplifies memory management through a two-line LLM wrapper that automates the persistence of agent experiences.

plastic-labs / honcho

Honcho is a sophisticated library that uses asynchronous reasoning to extract conclusions from message traces and event logs. Its peer-centric model tracks evolving entities and multi-peer perspectives, utilizing hybrid BM25 and vector search for nuanced context retrieval.

greyhaven-ai / autocontext

This recursive harness uses a specialized five-agent loop to iteratively improve context for complex tasks. It generates structured playbooks and traces to ensure knowledge persistence across sessions for Hermes and other LLM providers.

stephenschoettler / hermes-lcm

A lossless context management plugin that uses a hierarchical DAG structure to ensure no message is ever truly lost during context compaction. It provides agents with specialized tools to search, describe, and expand compressed history stored in a persistent SQLite backend.

AxDSan / mnemosyne

A zero-dependency memory system designed for sub-millisecond latency in read, write, and search operations. It functions as a native Hermes plugin, leveraging SQLite for local-first storage with integrated vector and full-text search.

elkimek / honcho-self-hosted

This project provides a self-hosted backend for the Honcho memory library, enabling private cross-session persistence for Hermes Agent. It includes automated Docker setup scripts and supports any OpenAI-compatible provider or local inference server.

yoloshii / ClawMem

ClawMem is a local-first context engine that operates entirely without cloud dependencies or API keys. It integrates via MCP or agent plugins, utilizing a hybrid retrieval stack of BM25, vector search, and knowledge graphs.

plur-ai / plur

Plur introduces a shared memory layer using an open YAML 'engram' format to facilitate zero-cost local storage. It implements ACT-R activation logic to strengthen relevant facts while allowing stale information to decay naturally over time.

EfficientContext / ContextPilot

Focused on performance, this tool optimizes long-context inference by reducing prompt tokens by up to 36% through deduplication. It significantly boosts prefill speeds for Hermes Agent and vLLM by utilizing optimized prefix sharing.

keepnotes-ai / keep

Keep provides reflective memory by indexing codebases, documents, and media with automatic OCR and summarization. It creates navigable, bidirectional relationships between notes using edge tags, accessible via MCP and CLI interfaces.

yepyhun / Brainstack

Brainstack consolidates temporal continuity, graph truth, and corpus retrieval into a single kernel stack for Hermes. It employs a unique 'five-shelf' architecture to organize data and prevent the common issue of flat, unorganized memory blobs.

amanning3390 / flowstate-qmd

This anticipatory memory system prefetches context by monitoring agent session logs to stay ahead of user needs. Built for the Hermes 2026 Hackathon, it uses SQLite-vec and GGUF models for a completely local-first, MCP-native experience.
