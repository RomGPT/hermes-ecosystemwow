breakdown

mem0ai / mem0

As the official memory provider for Hermes Agent, Mem0 offers a multi-level architecture that retains user, session, and agent states. It utilizes a token-efficient single-pass extraction algorithm and hybrid retrieval to fuse semantic, keyword, and entity-linked signals.

garrytan / gbrain

This opinionated 'agent brain' features a self-wiring knowledge graph with typed entity links for sophisticated relationship mapping. It supports hybrid vector and graph-based retrieval and connects via MCP, CLI, or OAuth 2.1 for secure remote operations.

vectorize-io / hindsight

Hindsight implements a 'retain, recall, and reflect' workflow that allows agents to learn and improve from past interactions over time. It provides a simple two-line LLM wrapper for automatic memory management and supports per-user isolation through custom metadata filtering.

plastic-labs / honcho

Honcho is a stateful memory library that maintains evolving context for users, agents, and groups across Python and TypeScript SDKs. It enables natural language querying of interaction histories and formats context to be natively compatible with OpenAI-style APIs.

greyhaven-ai / autocontext

This recursive context harness uses structured traces and real evaluation to iteratively improve agent performance on complex tasks. It facilitates multi-role cooperation for strategy refinement and knowledge curation across various model providers.

stephenschoettler / hermes-lcm

A specialized plugin for Hermes Agent that uses a hierarchical DAG structure to ensure lossless context management and depth-aware compaction. It includes a SQLite-backed store and dedicated tools for grepping or expanding compacted history without losing raw message data.

elkimek / honcho-self-hosted

This project provides a self-hosted backend for the Honcho library, enabling private cross-session persistence specifically for Hermes Agent users. It supports local inference servers and includes automated scripts for deploying a full stack with Docker, PostgreSQL, and Redis.

yoloshii / ClawMem

ClawMem is a local-first memory engine that operates entirely on-device without cloud dependencies or API keys. It employs a hybrid retrieval system combining vector search, BM25, and causal graph traversal to share memory across Hermes, Claude Code, and OpenClaw.

EfficientContext / ContextPilot

Focused on inference efficiency, ContextPilot optimizes long-context LLM performance through prefix reordering and redundant block deduplication. It natively integrates with Hermes Agent and vLLM to boost cache hits and reduce input token usage.

plur-ai / plur

Plur utilizes an open YAML 'engram' format to create a shared memory layer for multi-agent systems with zero API costs. It automates memory injection for Hermes and coding agents using a hybrid of BM25 and local embedding search.

keepnotes-ai / keep

Keep provides reflective memory by indexing codebases, documents, and media with automatic OCR and summarization. It uses edge tags to build bidirectional relationships between notes, accessible to agents via MCP and CLI interfaces.

AxDSan / mnemosyne

A zero-dependency memory system built on SQLite that offers sub-millisecond latency for local-first Hermes Agent deployments. It features native vector and full-text search alongside automated episodic memory consolidation.

amanning3390 / flowstate-qmd

Developed for the Hermes 2026 Hackathon, this anticipatory memory system prefetches context by monitoring agent session logs. It uses SQLite-vec and GGUF models for local-first storage and provides an MCP-native server for seamless client configuration.
