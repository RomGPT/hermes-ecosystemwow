breakdown

mem0ai / mem0

A universal memory layer and official Hermes Agent provider that manages user, session, and agent states. It utilizes hybrid retrieval through semantic, keyword, and entity matching and is available as a managed cloud or self-hosted Apache-2.0 project.

supermemoryai / supermemory

An official Hermes Agent memory engine featuring a custom vector graph engine and context fencing for high-performance recall. It maintains persistent memory graphs and automated user profiling, sustaining sub-300ms recall speeds for massive token volumes.

garrytan / gbrain

An opinionated agent brain that automatically extracts typed entity relationships into a self-wiring knowledge graph. It includes a synthesis layer for cited prose answers and supports scoped data access for multi-user company environments.

vectorize-io / hindsight

A learning-focused memory system that implements retain, recall, and reflect workflows for long-term agent improvement. It provides a simple two-line LLM wrapper and supports a wide range of providers, including OpenAI, Anthropic, and local models.

campfirein / byterover-cli

An official Hermes Agent provider that treats memory as a git repository, allowing for branching, merging, and versioning of context trees. It features a 5-tier retrieval system with sub-100ms latency and integrates with the Model Context Protocol.

plastic-labs / honcho

A stateful agent library that extracts reasoning-based conclusions from conversations rather than simple text chunks. It tracks multi-peer perspectives to model what different entities know about one another via a FastAPI-based server.

greyhaven-ai / autocontext

A recursive self-improving harness that distills successful agent strategies into reusable playbooks. It employs multi-role coordination for strategy curation and supports MCP server integration across multiple LLM providers.

AxDSan / mnemosyne

A zero-dependency memory system utilizing a SQLite backend to achieve sub-millisecond retrieval performance. It manages long-term knowledge through episodic compression and temporal triples, supporting Python SDKs and native agent plugins.

stephenschoettler / hermes-lcm

A lossless context management plugin that uses a hierarchical summary DAG for depth-aware compaction. It preserves raw messages in a SQLite store and provides specialized tools like lcm_grep and lcm_expand_query for precise retrieval.

elkimek / honcho-self-hosted

A self-hosted backend for the Honcho library that enables private, cross-session persistence for Hermes Agent. It supports any OpenAI-compatible or local inference server and includes automated Docker, PostgreSQL, and Redis setup scripts.

Signet-AI / signetai

A local-first identity and secrets layer that provides portable state across Hermes Agent, Claude Code, and MCP clients. It uses SQLite for storage and features a repairable system for editing or reclassifying agent context.

yoloshii / ClawMem

An on-device context engine that eliminates cloud dependencies and API keys for local-first operation. It employs hybrid retrieval combining BM25, vector search, and knowledge graphs via MCP servers and agent plugins.

plur-ai / plur

A shared memory layer using an open YAML engram format to enable multi-agent systems without API costs. It utilizes ACT-R activation to strengthen relevant facts while decaying stale information through MCP tools.

EfficientContext / ContextPilot

A context optimization tool that increases cache hits by 4–12× through prefix reordering and deduplication. It reduces prefill latency by 1.5–3× and provides native plugin support for the Hermes Agent and OpenClaw ecosystems.

yantrikos / yantrikdb-hermes-plugin

An embedded, self-maintaining plugin that provides explainable recall via why_retrieved tags. It handles contradiction tracking and canonicalization in-process, requiring no separate server or GPU.

yepyhun / Brainstack

A memory kernel stack that consolidates temporal continuity, graph truth, and corpus retrieval. It leverages a multi-layered storage approach using SQLite, Kuzu, and Chroma to provide bounded evidence packing and reduce prompt noise.

keepnotes-ai / keep

A reflective memory system combining BM25 full-text search with ranked graph traversal. It automatically indexes local directories, URLs, and multimedia files, using edge tags to create navigable relationships with inverse links.

amanning3390 / flowstate-qmd

An anticipatory memory system that prefetches context by monitoring agent session logs. It uses local-first storage with SQLite-vec and GGUF models, operating as an MCP-native server for coding agent clients.

alias8818 / hermes-tool-slimmer

A utility that reduces prompt token overhead by using local BM25 ranking to select only relevant tool schemas. It includes a dashboard to track token savings and is compatible with Hermes Agent v0.14.0.

Ladybug-Memory / hermes-memory-plugin

A local-only plugin using a columnar .lbdb embedded graph database and importance-weighted recall to surface high-priority memories. It supports BM25 keyword search and optional GLiNER2 entity extraction.

MukundaKatta / hermes-agentmemory

A pull-model episodic memory plugin that ensures complete deletion of memories without leaving persistent summaries. It provides a trace.jsonl audit log for every operation and utilizes the Claude model family for on-demand summarization.

futurebrowser / hermes-exabase-plugin

A plugin that integrates Exabase M-1 to provide self-organizing long-term memory. It offers tools for manual storage and searching, alongside configurable query expansion and result reranking.
